Week 9

More GUI Widgets

JFileChooser is a dialog box that lets the user pick a file. This widget is explored more in-depth in Week 7. JColorChooser is a dialog box that lets the user select a color from the palette. JMenuBar creates a top-level container that holds menus. In JFrame, we use this method:

public void setJMenuBar(JMenuBar bar)

Every JMenu in the JMenuBar is going to contain JMenuItems. Here are the JMenu methods:

  • public JMenu(String text)
  • public void add(JMenuItem item)
  • public void addSeparator()
  • public void setMnemonic(int mnemonic)
  • setToolTipText(String text)

JMenuItem can be instantiated with the text, icon, and mnemonic. The mnemonic is the key that can be pressed to trigger the menu item selection. Mnemonics will only work when the component is in focus; however, accelerators work even when the componenent is not in focus:

(KeyStroke.getKeyStroke('V', keyEvent.CTRL_Mask));

Remember that Windows and MacOS have different accelerators! For Windows, it should be alt. For MacOS, it should be option + ctrl.

JToolBar is a container for GUI components. The toolbar can be moved around within a border layout. To add an image to the button that is in a toolbar, you must scale it down appropriately.

GUI Component Methods

The setOpaque(boolean opaque) method sets a GUI component to be opaque, i.e., not transparent. Always make a frame’s content pane (frame.setContentPane(---)) opaque. revalidate() tells the layout manager to recalculate the layout. repaint() repaints the component to see new changes. Be sure to use these last two methods for Assignment 5.

2D Graphics

If you want to draw something on the GUI, you need to use the paintComponent method, which is present in every JComponent class. Think of the Jpanel as a piece of paper and the Graphics object as your paintbrush. Do not confuse paintComponent with paintComponents! Always call the superclass’s constructor. One Graphics object does all the drawing for the component. It is always best to cast a Graphics object as a Graphics2D object.

@Override
public void paintComponent(final Graphics theGraphics) {
    super.paintComponent(theGraphics);
    final Graphics2D g2d = (Graphics2D) theGraphics;

    g2d.setRenderingHint(RenderingHints.KEY_ANTIALIASING,
                         RenderingHints.VALUE_ANTIALIAS_ON);
}

The code is pretty boilerplate, so don’t be afraid to copy and paste.

Shapes are made like so:

final Shape ellipse = new Ellipse2D.Double(10, 10, 50, 100);
g2d.setPaint(Color.ORANGE);
g2d.setStroke(new BasicStroke(STROKE_WIDTH));
g2d.draw(ellipse);

Here are some shapes:

  • Rectangle2D.Double(x, y, height, width)
  • Ellipse2D.Double(x, y, height, width)
  • Line2D.Double(x1, y1, x2, y2)
  • Arc2D.Double(x, y, width, height, start, extend, int type)

There also exist .Float(---) methods. The Graphics2D object is what will paint the shapes. Remember that an ellipse’s dimensions are based on its bounding rectangle. You need to do Graphics2DObject.fill(Shape shape) if you want to fill the shape with a color; otherwise, only the stroke will be visible. BasicStroke is the constructor for the basic form of the Stroke class. Shape methods includes the following:

  • getBounds()
  • getCenterX() / getCenterY()
  • getMinX() / getMinY()
  • getMaxX() / getMaxY()
  • intersects(double x, double y, double w, double h)
  • intersects(Rectangle2D r) (which returns a boolean value)

A Path2D is a series of line segments or curves. Its methods include:

  • append(Shape s, boolean connect)
  • lineTo(float x, float y)
  • moveTo(float x, float y)

Other Important Notes

Don’t forget about JToggleButton.

group.clearSelection() clears the selection of a button group.

setText() is the method for changing the text of a component.

When you call repaint(), by default the paint component method is called, so don’t ever call repaint inside paint component. This will result in stack overflow.

String... in a parameter list means you can input any number of parameters.
