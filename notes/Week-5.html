Week 5

Review for Test 1

You can bring one textbook or one page of Java syntaxes. These should only contain the name of the method and its parameters (the method signatures). You can bring only one or the other, but not both.

Hashcode helps to determine where an object is stored. Imagine a hashmap as an array of objects that each point to “buckets.” The hashcode helps determine which “bucket” an object should be placed in.

For primitives, the == operator has the same effect as the equals method. Otherwise, the two are entirely different.

An array has a fixed size, but not an arraylist.

The containsValue() method of hashMap cheks if a given “value” is present in the hashmap. You cannot loop through a hashmap with a normal for loop. You have to use an iterator. Remember you cannot use the same iterator to iterate over the same map unless you re-initialize it.

The constructor method should be as simple as possible. It is best not to call any other methods. This is because the only purpose of the constructor is to initialize fields. Constructors do not always need to be private. A private constructor is used for utility classes. The very first statement of a child class constructor should be super(arguments).

Public members can be accessed anywhere, even outside the package. Protected members an be accessed outside the package only if it is in a sub-class.

Final classes cannot be extended. Final methods cannot be overridden.

Static members are tied to a class, but not the individual instances of the class. Inside a static method, you can only access the static fields. Static variables are common to all objects of the class.

All test cases are public methods whose declarations are preceded by the @Test tag. Methods that are executed before every test method are declared with the @Before. assertSame compares memory addresses while assertEquals uses the equals() method.

Be sure to cast the object when comparing them with the equals() method. equals() tests for four conditions:

  • If the two objects are the same instance.
  • If the two objects are the same class.
  • If the object that was passed is null.
  • If the two objects have the same field values.

Use Double.compare(a, b) to compare two doubles.

Remember that BigDecimal is much more accurate than double or float:

final BigDecimal bd1 = new BigDecimal("123.456");
bd1.setScale(3, RoundingMode.CEILING);

Inheritance

Java can only extend classes once. It does not support multiple inheritance. The child class does not inherit the constructor of the parent.

instanceOf checks if a particular class is a sub-class of a parent. It returns a boolean value. null instanceOf x always returns false.

The keyword super refers to the super-class. Members of a parent class need not be accessed with super. By default, Java calls the parameterless constructor of a parent class. If the parent class has a parameterized constructor and you do not call super, it will throw an error.

Method Overloading refers to writing multiple versions of a method that each have different parameters. Method Overriding refers to reimplementing a method of a parent class. Be sure to preced overridden methods with the tag @Override. The method signatures must be different for overridden methods.
