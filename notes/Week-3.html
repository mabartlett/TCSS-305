Week 3

Objects Class

java.util.Objects contains helpful methods for objects. Note that this is different from the class that is simply named Object. Object.equals(P1, P2) is the same as P1.equals(P2). Arrays.deepEquals(theObject1, theObject2) performs a deep equals operation, i.e., rather than comparing the two objects’ memory locations, the data contained in the two objects are compared. The function requireNonNull() of the Objects class verifies that the passed object is not null.

String Format

If you have a string you want to format, use the String.format() method. The first parameter is the locale, i.e., the country. For example, it could be Locale.UK or Locale.GERMAN. Remember to import java.util.Locale.

The second parameter is the forma, the string format.

The third parameter is the string itself being passed with some of the following string format specifiers:

  • %s string
  • %f floating point number
  • %.6f floating point number with 6 decimal points
  • %d integer
  • %15d integer of length 15 (spaces are prepended)
  • %015d integer of length 15 (zeroes are prepended)

The parameters after that are data being put into the string.

Number Format

Numbers can be formatted just like strings:

NumberFormat.getInstance(Locale.US).format(myNumber);

If you want to print it like a currency, use the following code:

NumberFormat.getCurrencyInstance(Locale.US);

Remember to import java.text.NumberFormat.

Big Decimal

When working with things like currencies, it is better to use BigDecimal. It is an arbitrary precision integer + . + 32 bit integeger representing the number of digits after the decimal point. It is more accurate than a float or double. It is declared like so:

BigDecimal bd1 = new BigDecimal("0.06");

Operations on a BigDecimal are performed by using methods of its class like these:

  • bd1.add(bd1)
  • bd1.subtract(bd1)
  • bd1.multiply(bd1)
  • bd1.divide(bd1)
  • bd1.pow(2)
  • bd1.negate()

BigDecimal.valueOf(bd) is an important method. Another one is bd1.setScale(3, RoundingMode.CEILING), The rounding mode can also be FLOOR.

Exception Handling

Exception indicates conditions that a reasonable application might try to catch. Checked exceptions occur at compiler time, such as FileNotFound Exception while reading a file. An unchecked exception occurs at execution time, such as ArrayIndexOutOfBounds Exception. Catching exceptions are done like so:

try {
    // Code
} catch (final Exception e) {
    System.out.println(e);
} catch (final IllegalArgumentException | NullPointerException e) {
    System.out.println(e);
} finally {
    // This code is always executed whether or not a catch block is executed
}

Some methods in the Exception class include:

  • getMessage()
  • printStackTrace()
  • toString()

Implicitly throwing an error involves using the Objects.requireNotNull() method, while explicitly throwing an exception uses the code throw new IllegalArgumentException(). Here’s an example:

this.myPoint = Objects.requireNonNull(theOtherObject).myPoint;

You can define your own exception class by extending Exception. Always be sure to include a toString() method.

Input and Output

Streams are sequences of data. Byte streams read/write 1 byte at a time and character streams read/write 2 bytes at at time. There are 3 standard streams taht are automatically opened (and closed) by the OS:

  • Standard Input (System.in)
  • Standard Output (System.out)
  • Standard Error (System.err)

System.err is used in almost the same way System.out is.

Other Important Notes

Any method of an object that is present in its parent class must be preceeded by the symbol @Override. This would be for methods like equals() or toString().

Final classes cannot be extended.

Whenever the constructor is private, the class is called a utility class.

Hash map items are not in any particular order.

Iterators must iterate a map’s key set, not the map itself.

Don’t surround the exceptions in assignment 1 with a try/catch block. Simply throw them.

FileOutputStream() has two parameters. The first is the file, and the second is a boolean value representing whether contents are being appended to the file.

final String workingDirectory = System.getProperty("user.dir");

Remember to close any input and output files! They must also be closed in the correct order.

BufferedWriter writes text. You can write more than one byte at a time with it. BufferedReader reads text in the same way.
