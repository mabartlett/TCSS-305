Week 8

Event Listening

Whenever a GUI component is attached to some functionality, we say that the button is listening to an action event. The events are in java.util.EventAction. There are specific action events tied to each components. getActionCommand() returns a String that represents the event. An ActionListener is an interface. It has one abstract method, which is actionPerformed(ActionEvent evt). You have to create a new class for the listener before adding it to a component.

Where should the listener class be defined? One solution is to use the GUI class as the ActionListener:

public class SomeGUIClass implements ActionListener {
    JButton button = new JButton("A Button");
    button.addActionListener(this);
}

Another solution is to use an inner class, which means you define a class within another. You can only instantiate an object of the inner class after you have instantiated an object of the outer class:

InnerClass innerObject = outerObject.new InnerClass(params)

You can also define a local inner class, which is when you define a class from inside a function. An anonymous inner class is defined without a name:

button.addActionListener(new ActionListener() {
    @Override
    public void actionPerformed(final ActionEvent theEvent) {
        theObject.doSomething();
    }
});

Mouse Listeners

MouseListener is the interface we implement to listen to MouseEvents:

  • mouseClicked()
  • mouseEntered()
  • mouseExited()
  • mousePressed()
  • mouseReleased()

MouseAdapter is a class that implements all five methods so you do not have to implement all five methods yourself. This saves you a lot of trouble!

mouseDragged() and mouseMoved() are defined in a different interface called MouseMotionListener. There also exists the MouseMotionAdapter class which implements the mouse motion interface. For the mouse’s scroll-wheel, there is the MouseWheelListener interface, which declares the mouseWheelMoved() listener function. MouseInputListener is an interface which contains all the methods inside MouseListener and MouseMotionListener. The best class to implement all three methods is MouseInputAdapter. This class is the king of them all because it provides empty implementations for all eight methods.

Keyboard Listeners

Keyboard listening is much simpler than the mouse:

  • keyPressed()
  • keyReleased()
  • keyTyped()

The class that implements all three methods is the KeyAdapter class.

KeyEvent has public static fields, key codes, for every key that can be pressed. For example, VK_A,…, VK_Z represent the key codes for the letters. Every keyboard can have its own unique key codes. KeyEvent contains the following methods:

  • getKeyChar()
  • getKeyCode()
  • getModifiers()
  • isActionKey()

Review for Test 2

The comparable interface provides natural ordering for objects and allows for sorting. You must override the public int compareTo(Object obj) method in classes that implement comparable. Another important interface is cloneable.

An encapsulation violation is when a client class is able to modify the fields directly of another class. To make a copy of an instance, you must clone all the mutable fields by utilizing a defensive copy routine. The method that clones is called clone(). Be sure to cast the objects as the appropriate classes. An array or list of any object is always mutable. Immutable objects include but are not limited to:

  • String
  • All eight wrapper classes
  • Enum
  • BigDecimal
  • Color

Even if there is not an @Override tag, a method may still be overridden.

Do not forget that Java does not support multiple inheritance. Do not confuse overridden and overloaded methods. Overloaded method calls first look at the method signature. Then comes the binding. An overloaded function will always call the function attached to the data-type.

The default layout for JFrame is border, while the default layout for a panel is flow.
