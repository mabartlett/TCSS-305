Week 6

Static and Dynamic Binding

Static binding is for overloaded methods while dynamic binding is for overridden methods. Always keep in mind what class each object is. An instance of a child object can be declared as a type of the parent object. Whenever there are overloaded methods, you need to call the method that belongs to the type of the variable. Remember that overloaded methods will be called based on the parameters passed, regardless of whichever class is being called. If you declare an object with a parent class’s data type, but use the child class’s constructor, that object will only have access to the members of the parent class.

You can always cast between hierarchies.

If an object variable is declared as a parent class’s type, but is initialized with the child class’s constructor and the child class has an overridden method, when calling that method, the child class’s type will be used. Overloaded methods are called based on the object’s constructor while overridden methods are called based on the object’s declared data type.

All that can be confusing, I know.

Interfaces

Interfaces are classes that contain only abstract methods. Interfaces are implemented by another class. The implementing class must define the methods that were declared in the interface.

Interfaces are defined with the keyword interface. Using the abstract keyword before each method of an interface is redundant. Do not use it. By default, fields are public, static, and final, so using the keywords public, static, and final is redundant.

There are two other types of methods that can be a part of the interface. The first is a static method. The other is the default method. Both can be implemented. These methods cannot be reimplemented by the implementing class. You cannot create an object of an interface. You can only instantiate an object of an implementing class.

Remember to use the @Override tag before each method’s definition in the implementing class.

Interfaces were introduced because Java only supports single inheritance. You can implement more than one interface. When you implement more than one interface, be sure to define all the methods from all of the interfaces.
