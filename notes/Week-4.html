Week 4

Testing

There do not exist “software testers” anymore. Testing is the developer’s job.

There are three main kinds of testing:

  • Unit testing
  • Integration testing
  • System testing (which can be devided into function and performance testing)

Blackbox testing tests only input and output while whitebox testing tests each line and focuses on internal states of objects. Another way of choosing tests is path testing, which is trying to use test input that will pass through each path in the code. This is a form of whitebox testing.

Unit testing is done using JUnit. The version used in class is version 4, but this is not the current version. Test cases are made by making methods that are preceded by specific annotations, which can include the following:

  • @Test marks an instance method as a test case. They must be public.
  • @Before marks an instance method so that it gets run once before every test method is run.
  • @After marks an instance method so that it gets to run once after every test method is run.

Import JUnit with

import static org.junit.Assert.*;

Follow this procedure when writing a test class:

  1. Declare a private instance of the object.
  2. Initialize it in the setup method.
  3. Write the test case.

When testing for an exception, state this with the @Test tag:

@Test(expected = IllegalArgumentException.class)

To write a test case for an equals method, you would actually write two methods. The first is if the method returns positive and the second is if the method returns negative. It returns positive if the object being compared is itself or if the object has the same fields. It returns negative if the object being compared is null, if it has different values, or if the objects are different classes.

When testing the hashCode method, check if it is equal to the Objects.hash output. Also test that, if the objects are equal, the hashcodes are equal.

Assertions

An assertion can either pass or fail. If it passes, the method returns with no effect. If it fails, an error is thrown.

assertEquals("Fail message", Object expected, Object actual) takes two objects or types and checks to see if they are equivalent. If you want to check two floats, you can pass a third double as an argument, which is the “tolerance”, i.e., the allowed distance between the two floats. Here are some other assertion methods:

  • assertNotSame
  • assertNull
  • assertTrue
  • assertFalse
  • fail

When writing test cases, do not make them too simple. You must try to cover all possible cases. The test cases should cover 100% of the class being tested. Each test case should only test one functionality. All test cases should have Javadoc.
